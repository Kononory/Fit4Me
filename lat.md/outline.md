# Outline

Indented-text outline format, parser, and the in-app outline editor panel.

## Outline Parser
Converts indented plain text into a [[tree#Tree]] and back. It is the primary way to bulk-edit a [[flow#Flow]]'s structure — used both when creating a new flow from text and inside the outline editor panel.

Source: `src/parser.ts` — `parseOutline()`, `treeToOutline()`

## Outline Format
Each line is one node; 2-space indentation defines parent-child relationships.

```
Root Label [root]
  Nav Label [nav] | Tab A · Tab B
    Tab A [tab:branch-a]
      Screen 1
      Screen 2 | optional sublabel
```

Line syntax: `Label [type:branch] | sublabel`. Node IDs are auto-generated by slugifying the label; duplicates get a numeric suffix (`label-2`).

`->` arrow prefix is an alternative to 2-space indentation: `->Child` = 1 level, `->->Child` = 2 levels. Works both when pasting and in real-time as you type. Normalized by `normalizeArrows()` before parsing.

## Branch Propagation
After parsing, `propagateBranch()` copies each `tab` node's branch ID down to all its descendants. This ensures [[canvas#Canvas]] branch highlighting works without the user annotating every line.

## Outline Editor Panel
Opened with `⌘E`. Pre-filled with `treeToOutline()` output. On apply (`⌘↵`), re-parses and replaces the active tree.

Keyboard shortcuts inside the panel: `Tab` / `Shift+Tab` — indent/unindent. `->` — arrow prefix auto-converts to one indent level. `Shift+Enter` — new line. `⌘Enter` — apply. `Esc` — discard.

Source: `src/components/TextEditPanel.tsx`
